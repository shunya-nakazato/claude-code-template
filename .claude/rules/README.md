# .claude/rules/

プロジェクト開始時に固め、開発期間を通じて本質的に変わらない方針・規約を管理するディレクトリ。

## 目的

- Claude Code がコード生成・レビュー時に参照する**不変の基準**を定義する
- CLAUDE.md から `@.claude/rules/<file>.md` で参照される

## 含めるべき内容

| カテゴリ | ファイル例 | 内容 |
|---|---|---|
| プロジェクト概要 | `project-info.md` | プロジェクト名、概要、対象ユーザー、リポジトリ構成 |
| 技術スタック | `tech-stack.md` | 言語、フレームワーク、インフラ等の技術選定 |
| コーディングスタイル | `code-style.md` | 命名規則、ディレクトリ構成、コメントスタイル |
| コーディング規約 | `code-rule.md` | 言語・フレームワーク固有のコーディングルール |
| Git運用 | `git-rule.md` | コミットメッセージ規約、ブランチ戦略 |
| CI/CD | `cicd-rule.md` | パイプライン構成、環境構成、デプロイ手順 |
| テスト戦略 | `test-rule.md` | テスト種別、ツール、カバレッジ基準 |
| セキュリティ | `security-mgmt.md` | バリデーション、インジェクション対策、機密情報管理 |
| エラー・ログ | `error-logging.md` | エラー分類、ログレベル基準、ログフォーマット |
| 環境変数・設定 | `env-config.md` | `.env`ファイル階層、命名規則、バリデーション方針 |

## 含めるべきでない内容

- **頻繁に更新されるドキュメント** — API仕様、設計ドキュメント等は `docs/` で管理する
- **一時的なメモ** — タスク固有の情報は `.claude/plans/` や issue で管理する
- **環境固有の設定値** — 実際の `.env` ファイルや secrets はここに置かない

## 運用ルール

- 各ファイルは setupスキル実行時にプロジェクトの実態に合わせて記入する
- テンプレートの冒頭にある `< 以下の内容を参考とし、プロジェクトの実態に基づいて更新すること >` は記入後に削除する
- 新しいルールカテゴリが必要な場合はファイルを追加し、CLAUDE.md に参照を追記する
