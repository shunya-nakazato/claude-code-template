# Lessons

## 1. ワークフローは例外なく守る
- タスクの大小に関わらず: todo.md → plans/ → /plan-review → 実装
- コンテキスト復元後は特に注意。例外はユーザーが明示的に許可した場合のみ

## 2. プランファイルの命名規則を守る
- `plan-<YYYYMMDDTHHmmssZ>-<概要>.md` に即座にリネームする

## 3. プランの粒度は 1プラン = 1ステップ
- todoの各ステップごとにプラン作成 → 実行 → 検証 → 完了マークのサイクルを回す

## 4. Bash コマンドでは相対パスを使う【最重要】
- 絶対パス禁止。例外なし。毎回確認すること

## 5. プランファイルの編集には Bash を使う
- `.claude/plans/` 配下は `cat >` / `cat >>` で編集し、承認待ちを回避する

## 6. Codex FAIL の繰り返しに固執しない
- 3回連続 FAIL で致命的問題がなければ独自判断で進行する

## 7. ユーザーからプランが提供されても後続タスクでプランを省略しない
- UI調整・スタイル修正でもプラン作成は必須。例外はユーザーの明示的許可のみ

## 8. セッション継続時にプランを再作成しない
- `/continue` で再開し、既存プランから即座に実装開始。再検討はユーザー依頼時のみ

## 9. デバッグは仮説駆動で行う
- 散弾銃デバッグ禁止。症状把握 → 仮説立案 → 修正の順で進める
