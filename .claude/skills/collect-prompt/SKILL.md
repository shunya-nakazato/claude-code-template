# Skill: @prompt ディレクティブ収集

## 概要
ソースコード内の `@prompt` ディレクティブを差分ベースで収集し、`.claude/tasks/todo.md` にタスクとして登録するスキル。

## トリガー
ユーザーが `/collect-prompt` を実行した時

## プロンプト記法

開発者はソースコード内のコメントブロックに `@prompt` を記述して指示を埋め込む。

```
/**
 * @brief 関数の概要説明
 * @prompt ここにClaude Codeへの指示を自然言語で記述する。
 * 複数行にわたって記述可能。
 */
```

## フロー

### Step 1: 差分検出

1. `git diff HEAD` で差分のあるファイルを確認する
   - ステージング済み・未ステージングの両方を確認する場合は `git diff` と `git diff --cached` を併用する
   - 差分が存在しない場合は `git diff HEAD~1` で直前コミットとの差分を確認する

### Step 2: @prompt 収集

2. 差分のあるファイルを対象に、すべての `@prompt` ディレクティブを集計する
   - 変更のないファイルに含まれる `@prompt` は**対象外**とする（差分駆動）
   - 同一ファイル内に複数の `@prompt` がある場合は出現順にリストする
   - 依存関係がある場合（例: 関数Aの結果を関数Bが使う）は依存順を注記する

### Step 3: 収集結果の提示

3. 収集した `@prompt` 一覧をユーザーに提示する:

```markdown
## 収集した @prompt 一覧

| # | ファイル | 対象 | 指示概要 |
|---|---------|------|---------|
| 1 | src/example.ts:12 | test() | hello worldを出力する関数を作成 |
```

### Step 4: tasks/todo.md への登録

4. ユーザーの承認を得た上で、収集した `@prompt` を `.claude/tasks/todo.md` にタスクとして追記する
5. 既存の `.claude/tasks/todo.md` がある場合は末尾に追記し、ない場合は新規作成する
6. 各タスクには元の `@prompt` の場所（ファイル:行番号）を参照として残す

```markdown
## @prompt から生成されたタスク (YYYY-MM-DD)

- [ ] test() — hello worldを出力する関数を作成 (src/example.ts:12)
- [ ] ...
```

## ルール
- このスキルは収集とタスク登録のみを行い、実装は行わない
- タスク登録後、各タスクの実装はCLAUDE.mdのタスク管理手順に従う
