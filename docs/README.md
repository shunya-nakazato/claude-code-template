# docs/

プロジェクトの進行に伴い成長・更新されるドキュメントを管理するディレクトリ。

## 目的

- 開発中に蓄積される設計知識や仕様を体系的に整理する
- `.claude/rules/`（不変の方針）とは異なり、プロジェクトの成長に合わせて継続的に更新する

## 含めるべき内容

| カテゴリ | ファイル例 | 内容 |
|---|---|---|
| API仕様 | `api-design.md` | URL命名規則、バージョニング戦略、共通レスポンス形式、エンドポイント一覧 |
| アーキテクチャ | `architecture.md` | システム構成図、モジュール間の依存関係、データフロー |
| データモデル | `data-model.md` | DB スキーマ、エンティティ関連図、コレクション設計 |
| 外部連携 | `integrations.md` | 外部API・サービスとの連携仕様、認証フロー |
| 運用手順 | `operations.md` | 障害対応手順、ログ調査方法、定期メンテナンス手順 |
| 意思決定記録 | `decisions/` | ADR（Architecture Decision Records）形式での設計判断の記録 |

## 含めるべきでない内容

- **不変の方針・規約** — コーディングスタイル、命名規則等は `.claude/rules/` で管理する
- **タスク管理** — issue や `.claude/plans/` で管理する
- **機密情報** — APIキー、パスワード等は絶対に含めない

## 運用ルール

- 新しい機能やエンドポイントを追加した際は、関連するドキュメントも同時に更新する
- ファイル名は `kebab-case.md` とする
- 図表が必要な場合は Mermaid 記法を使用し、画像ファイルの追加を避ける
